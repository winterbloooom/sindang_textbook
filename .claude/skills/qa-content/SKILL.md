# qa-content — Sindang content.md 원고 검수

`textbook/{subject}/{part}/{chapter}/content.md` 초안을 HTML로 변환하기 전에 내용 품질을 점검한다. 리포트만 생성하고, **파일은 수정하지 않는다** (사용자가 승인 후 직접 수정하거나 별도 요청 시에만 수정).

## 언제 쓰는가

사용자가 다음과 같이 요청할 때:
- "이 챕터 QA 해줘" / "content.md 점검해줘" / "검수해줘" / "리뷰해줘"
- HTML 변환 직전 원고 품질 확인

챕터 경로가 명시되지 않으면 어느 챕터를 검수할지 먼저 물어본다. IDE에 열려 있는 content.md가 있다면 그것을 후보로 제안한다.

## 입력

- **대상**: `textbook/{subject}/{part}/{chapter}/content.md`
- **참고**: 필요시 동일 챕터의 `probs/questions.json` (난이도·용어 수준 참고용, 선택)

## 점검 항목

다음 4가지를 **한 번에** 수행하고, 섹션별로 구분된 리포트를 출력한다.

### 1. 문장 종결 온점 누락 (자동 수정 제안)

한국어 평서문 종결어미(`다`, `요`, `죠`, `네`, `까`, `니다`, `습니다` 등)로 끝나지만 `.`/`?`/`!`/`:` 등 문장부호가 없는 줄을 찾는다.

**제외 조건:**
- 프론트매터(`---` 사이) 내부
- 표 셀(`|` 구분), 리스트 항목이지만 한 줄에 명사구·짧은 키워드만 있는 경우
- `<그림: ...>`, `<box: ...>...</box>`, `예)` 같은 템플릿 마커 라인
- 제목(`#`, `##` ...) 및 코드·수식 블록
- 이미 온점이 있거나 줄 끝이 `)`/`」`/`"` 등 따옴표·괄호로 닫혀 있고 그 안쪽에 문장부호가 있는 경우
- 문장 중간에서 다음 줄로 이어지는 경우 (다음 줄이 빈 줄이 아니고 들여쓰기/조사로 연결되는 정황)

**출력 형식:** `line N: <원문>` → `<수정 제안>` 형태로 나열. 이 항목은 `bash: sed` 같은 일괄 수정 스크립트도 함께 제시 가능.

### 2. 시각 자료(그림) 추가 제안

본문에서 **그림이 있으면 이해가 크게 도움이 될** 부분을 찾는다. 현재 `<그림: ...>`이 없는 지점 중:
- 공간적·구조적 설명 (배치, 모양, 주기율표의 위치 등)
- 과정·변화 (반응 단계, 상태 변화)
- 비교·대조 (두 개 이상의 대상 차이 설명)
- 숫자·데이터 패턴 (주기성, 경향)
- 추상적 개념을 구체 예시로 연결해야 하는 지점

**출력 형식:** `line N 부근: "<해당 본문 발췌>" → 제안: <그림: 제목안>` + 한 줄 근거. 이미 `<그림: ...>`이 있는 문단은 제외한다.

### 3. 고령 학습자 기준 난이도·설명 부족 점검

대상 독자는 **정규 중등교육을 받지 못한 성인 학습자(고령층 포함)**. 다음을 찾는다:

- **선행 개념 없이 등장하는 전문용어** — 첫 등장에 괄호 풀이나 비유 설명이 없는 경우
- **단계 생략** — "~이므로 ~이다" 식으로 결론만 있고 중간 추론이 빠진 곳
- **비유·일상 예시 부재** — 추상 개념인데 학습자 경험과 연결할 실생활 비유가 없는 곳
- **한자어 밀도 과다** — 한 문단에 한자어가 연속되어 호흡이 어려운 곳
- **지시어·생략 과다** — "그것", "이러한" 등 지시어가 무엇을 가리키는지 즉시 복원 어려운 곳

**출력 형식:** `line N, "<발췌>" — 이유: <무엇이 어려운지> / 제안: <어떻게 풀어 쓰면 좋을지 한 문장>`.

판단이 애매하면 포함시키고 "검토 필요"로 표시한다. 과잉 지적보다 누락이 더 위험하다.

### 5. 내부 그림(SVG/HTML) 검증

`content.md` 또는 동일 챕터의 `figures/` 내 **인라인 SVG 또는 HTML 구조로 작성된 그림**을 검증한다 (래스터 이미지 `.jpg`/`.png`는 제외). content.md에 `<svg>` 태그가 직접 포함되어 있거나 `<box>`/HTML 블록으로 도식이 그려진 경우 모두 대상.

핵심 기준: **인쇄(A4) 환경 + 노안 성인 학습자**가 부담 없이 읽을 수 있는가.

각 그림마다 다음을 점검한다:

1. **최소 글자 크기** — 그림 내부 모든 텍스트가 본문 기본 크기의 **0.9배 이상** (본문 기본 13pt 기준 ≥ 11.7pt ≈ 11.7px). `<text font-size>`, CSS `font-size`, 상속되는 부모 값 모두 계산. 미달 시 해당 요소의 라벨과 현재 크기 보고.
2. **도형 겹침** — `<rect>`, `<circle>`, `<path>`, `<text>` 등 요소의 바운딩 박스가 **의도치 않게** 겹치는지. 의도된 겹침(레이어, 강조 박스 위 텍스트 등)은 제외. 특히 텍스트가 도형 경계·다른 텍스트와 겹쳐 가독성이 떨어지는 경우 반드시 보고.
3. **선 굵기** — `stroke-width`가 0.75pt(≈1px @ 96dpi) 미만인 선은 인쇄 시 소실 위험. 해당 요소 나열.
4. **색상 대비** — 텍스트·주요 선의 배경 대비비가 **WCAG AA (4.5:1)** 미만이면 보고. 계산은 상대 휘도(relative luminance) 기준.
5. **색상 의존성** — 두 가지 이상 대상을 **색으로만** 구분하는가. 패턴(점선/실선)·라벨·도형 모양 등 색 외 구분 수단이 병행되는지 확인. 색맹 및 흑백 인쇄 대비.
6. **라벨-대상 연결** — 라벨이 가리키는 도형과 시각적으로 멀거나, 리더선(leader line)이 다른 요소와 교차해 어느 것을 가리키는지 모호한 경우.
7. **프로젝트 규칙 위반** — `RULES_design.md` §1 인쇄 제약 위반: 그라디언트(`linearGradient`/`radialGradient`), 그림자(`filter: drop-shadow`, `box-shadow`), 어두운 배경(#000 근처 fill), 형광/채도 과다 색상.
8. **텍스트 래스터화 여부** — SVG 안 글자가 `<text>`가 아니라 `<path>`(outline)로만 존재하면 보고. 확대·선명도·검색성 저하 + 폰트 교체 불가.
9. **여백·밀도** — 그림 영역(viewBox) 대비 요소들이 너무 빽빽해 숨 쉴 공간이 없는지 정성 평가.
10. **화살표 일관성** — 흐름/인과를 나타내는 화살표의 방향·머리 스타일·굵기가 그림 내에서 일관되는지.
11. **본문 용어와 라벨 일치** — 그림 라벨이 본문에서 쓰인 용어와 철자·표기가 동일한지 (예: 본문 "전자껍질" vs 그림 "전자 껍질").
12. **차트 필수 요소** — 막대/선 차트라면 축 라벨, 단위, 범례가 있는지.
13. **그림 크기** — 인쇄 시 가로 너비 **60mm 미만**으로 렌더링되도록 설정된 경우 경고 (viewBox·width·컨테이너 기준 추정).

**출력 형식:**
```
### 그림 #{순번} — <그림 식별자 또는 line N>
- [❌ 글자 크기] <text> "주기" font-size=10px (기준 11.7px 미만)
- [⚠ 겹침] <circle cx=.. cy=..>가 <text> "원자핵"과 겹침
- [⚠ 대비] 라벨 #666 on #eee = 3.1:1 (AA 4.5:1 미달)
- [✅ 그 외] 통과
```
문제 없는 그림은 한 줄로 `그림 #N: 통과`.

판단이 수치적으로 불가능한 항목(여백 밀도, 화살표 일관성 등)은 "검토 필요" 태그로 포함한다.

### 4. 미처리 TODO 탐색

본문 내 다음 패턴:
- `TODO`, `todo`, `FIXME`, `XXX`, `?`, `[?]`, `(확인)`, `(추가)`, `(보완)`, `<!-- ... -->` 주석
- 빈 섹션 (제목만 있고 내용이 한 줄도 없는 `#`/`##`)
- 빈 `<그림: >` (제목 누락)
- 완성되지 않은 문장 (`...`, `…`으로 끝나거나 조사로 끝나는 줄)

**출력 형식:** `line N: <원문>` + 간단 분류(미완결 문장 / 빈 섹션 / 명시적 TODO 등).

## 절차

1. **대상 확인.** 사용자가 지정한 경로 또는 IDE에 열린 `content.md`를 읽는다. 없으면 묻는다.
2. **파일 전체 Read.** 줄번호가 포함된 상태로 읽어 이후 리포트에 line N을 정확히 참조할 수 있게 한다.
3. **5개 항목 순차 분석.** 각 항목은 독립적으로 수행. 파일을 수정하지 않는다.
4. **리포트 출력.** 아래 형식을 따른다:

```
# QA 리포트 — {chapter path}

## 1. 온점 누락 ({n}건)
- line 12: "원소는 물질의 기본 단위이다" → "원소는 물질의 기본 단위이다."
(없으면 "해당 없음")

## 2. 그림 추가 제안 ({n}건)
- line 45 부근: "주기율표의 1족은 ..." → <그림: 주기율표 1족 배치>
  근거: 위치 설명은 시각화 없이 이해하기 어려움

## 3. 고령 학습자 기준 보완 필요 ({n}건)
- line 30, "전자껍질" — 이유: 첫 등장인데 풀이 없음 / 제안: "(원자핵 주위를 도는 전자들의 층)" 추가

## 4. 미처리 TODO ({n}건)
- line 58: "TODO: 실생활 예시 추가"
- line 72: 빈 `<그림: >` — 제목 누락

## 5. 내부 그림(SVG/HTML) 검증 ({n}개 그림)
### 그림 #1 — line 95, <svg viewBox="0 0 400 300">
- [❌ 글자 크기] <text> "주기" font-size=10px (기준 11.7px 미만)
- [⚠ 겹침] <text "원자핵">이 <circle>과 겹침
- [⚠ 대비] 라벨 #666 on #eee = 3.1:1 (AA 4.5:1 미달)
### 그림 #2: 통과

## 요약
- 반드시 고칠 것: {온점 누락 n건 등 자동 수정 가능 건수}
- 사람 판단 필요: {나머지 n건}
- 다음 단계 제안: 우선순위 top 3
```

5. **행동 지침 확인.** 리포트 마지막에 "자동 수정 가능한 온점 누락만 바로 반영할까요? 나머지는 직접 검토 후 알려주세요." 형태로 묻는다. **사용자 승인 없이 content.md를 수정하지 않는다.**

## 원칙

- **수정 없음이 기본.** 이 스킬은 리포트 전용. 사용자가 명시적으로 "반영해줘"라고 할 때만 Edit 한다.
- **과잉 지적보다 누락 경계.** 특히 항목 3은 판단이 주관적이므로, 애매하면 "검토 필요" 태그로 포함.
- **근거와 함께 제시.** 각 지적은 왜 문제인지 한 줄 근거를 붙인다. 학습자 관점("이 단어는 처음 나오는데 풀이가 없음")으로 설명.
- **line N을 정확히.** 사용자가 바로 해당 줄로 점프할 수 있도록.
- **기출문제 난이도 참고.** 같은 챕터 `probs/questions.json`이 있으면 거기 용어·난이도를 본문이 감당하는지 가볍게 교차 확인(필수는 아님).

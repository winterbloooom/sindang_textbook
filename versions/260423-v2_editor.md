# Version 2 — 웹 기반 WYSIWYG 에디터 (계획)

- 작성일: 2026-04-23
- 상태: 계획 (구현 전)
- 이전 버전: [v1 — MD → HTML 변환 방식](260423-v1_md_to_html.md)

## 동기

v1은 "규칙 기반 MD → HTML 변환"이었다. 세 가지 한계가 확인됐다.
1. 규칙 + 에이전트 센스 조합이라 한 번에 깔끔히 전사되지 않음
2. 변환 1회당 토큰 비용이 큼
3. 사소한 수정도 Claude Code 호출이 필요함

v2는 변환 단계를 아예 없애고, **작성=편집=최종 형태**가 되게 한다. 작성자는 노션/옵시디언처럼 쓰고, 결과는 곧 인쇄용 A4 페이지다.

## 확정된 결정

| 항목 | 결정 |
|---|---|
| 레이아웃 모델 | flow 중심 + 제한적 드래그 (블록 순서 재정렬, 그림/박스 크기·정렬) |
| 저장 포맷 | 장별 JSON 블록 트리 1파일 |
| v1 병행 | 안 함 (v1 자료는 git 히스토리로만 남김) |
| PDF export | 브라우저 print → PDF |
| QA 내장 | 하지 않음 |
| 페이지 오버플로우 | **워드식**: 종이 크기 넘어가면 그대로 잘려 넘어감 |
| 페이지 추가 | 현재 페이지가 덜 찼어도 새 페이지 강제 추가 가능 |
| 시작 쪽번호 | 장별로 지정 가능 (기존 프런트매터 `start_page_num` 역할) |
| 저장 방식 | **실시간 자동 저장** — 편집 즉시 로컬 파일에 동기화 |
| 섹션별 레이아웃 | 이론 / 대표 기출 문제 / 기출 문제로 응용하기 — 세 가지 레이아웃 프리셋 전환 |
| 지원 브라우저 | **크롬 전용** (File System Access API 등 최신 API 자유 사용, 폴백 없음) |

## 기술 스택

- **에디터 코어**: TipTap (ProseMirror 기반). 커스텀 노드 자유도 때문에 선정.
- **프레임워크**: Vite + React, 로컬 우선.
- **저장 / 동기화**: File System Access API로 장별 JSON 파일에 직접 쓰기. 입력 발생 시 디바운스 후 즉시 쓰기 — "저장" 버튼 없음.
- **수식**: KaTeX. 인라인 `$...$` / 블록 `$$...$$` LaTeX 입력 → 실시간 렌더.
- **렌더러**: 에디터 뷰와 export 뷰 동일 React 컴포넌트 트리. 인쇄 CSS(`@page A4`)를 공유.

## 필수 블록 (커스텀 노드)

기존 템플릿에서 이미 확정된 요소들을 에디터 블록으로 이식한다.
- 문단 / **bold**(=`.hl`) / 번호매김 `ol1~ol4`
- `box` 변종(`.bx-*`) — 삽입 버튼에서 종류 선택
- `figure` — 이미지 파일 + 캡션, 파일 누락 시 `.fig-ph` fallback 자동 표시
- **표** — 셀 병합/분할, 셀·행·열 구분선 on/off(투명화), 정렬 지정
- 페이지 번호 — 자동 증가, **시작값만 장 속성으로 지정**
- 챕터 타이틀/셰브론 — 첫 페이지 전용
- **수식** — LaTeX 입력 UI (인라인/블록)

## 페이지 동작 (워드식)

- 페이지는 A4로 고정 렌더, 경계는 실시간으로 보임.
- 내용이 페이지 높이를 넘으면 **그대로 넘어감** — 자동으로 다음 페이지로 흘러감.
  - 자동 분할의 정확한 지점은 사용자 책임. 원하면 수동으로 페이지 나눔 삽입.
- 페이지가 덜 차도 **"새 페이지 추가" 버튼**으로 강제 추가 가능.
- 장 속성에 `start_page_num`을 지정하면 첫 페이지 번호가 그 값부터 시작, 이후 자동 증가.

## 섹션별 페이지 레이아웃

한 장은 본문 흐름 도중 **기출 문제 풀이 영역**으로 전환되며, 이때 페이지 레이아웃이 달라진다. 에디터는 섹션 종류를 블록 속성으로 인식하고 해당 페이지/컨테이너에 레이아웃 프리셋을 자동 적용한다. 페이지 번호 연속성은 레이아웃 전환과 무관하게 유지된다.

| 섹션 | 레이아웃 | 동작 |
|---|---|---|
| **이론** (`# 이론` 계열) | 단일 단 플로우 | 일반 본문. 기본 동작. |
| **대표 기출 문제** (`.bx-prob`) | 이론 **마지막 페이지 하단에 밀착** | 이론 블록과 같은 페이지 끝에 자동 pin. 페이지가 덜 찼어도 하단으로 내려감(flex-column + margin-top:auto). |
| **기출 문제로 응용하기** (`.apply-*`) | **별도 페이지 + 2단 그리드** | 문제/해설을 좌·우 두 열로 배치. 이 섹션이 시작되면 새 페이지로 강제 전환. |

섹션 전환은 슬래시 커맨드(예: `/기출문제풀이`) 또는 섹션 블록 삽입으로 트리거.

## UX 원칙

- 노션식 `/` 슬래시 커맨드 + 상단 툴바 병행.
- 이미지: 드래그앤드롭 → 해당 장 `figures/`에 자동 복사 후 삽입.
- 블록 드래그 재정렬 시 PPT식 정렬 가이드(페이지 마진, 블록 상하단 스냅).
- 자유 좌표 이동은 제공하지 않음 (인쇄 flow 문서 전제).

## 표 편집

- 셀 병합 / 분할.
- 셀 테두리 on/off 및 투명화(=구분선 제거 효과).
- 셀 내용 정렬(좌/중/우, 상/중/하).
- 행/열 추가·삭제.
- 기존 규칙 유지: `box` 내부 표는 `.tbl-wrap` 래핑하지 않음 (에디터가 컨텍스트로 자동 판단).

## 수식

- 입력: LaTeX 문법을 슬래시 커맨드 또는 `$...$` / `$$...$$` 트리거로.
- 렌더: KaTeX, 편집 중에도 실시간.
- 저장: JSON에 원본 LaTeX 문자열 보존.

## Export

- HTML: 에디터의 렌더 트리를 정적 HTML로 덤프. 기존 `RULES_design.md` CSS를 그대로 사용해 시각적으로 v1 결과물과 동등.
- PDF: 브라우저 print (Cmd/Ctrl+P) → "PDF로 저장". 별도 PDF 엔진 없음.

## 로드맵

1. **Phase 0 — 프로토타입**: TipTap 셋업, 문단/번호/굵게/박스 1종, A4 페이지 경계 렌더, 페이지 오버플로우 잘림 동작.
2. **Phase 1 — 그림·수식**: 이미지 드래그앤드롭, `.fig-ph` fallback, KaTeX 수식.
3. **Phase 2 — 표**: 기본 표 + 셀 병합, 구분선 on/off, 정렬.
4. **Phase 3 — 배치·페이지**: 블록 드래그 재정렬, 정렬 가이드, 수동 페이지 나눔 / 새 페이지 추가, `start_page_num` 장 속성, 실시간 자동 저장.
5. **Phase 4 — 섹션 레이아웃**: 이론 / 대표 기출 문제(하단 밀착) / 기출 문제로 응용하기(별도 페이지·2단 그리드) 프리셋.
6. **Phase 5 — Export**: HTML 정적 덤프, 브라우저 print 친화 CSS 정리.

## v1 대비 기대효과 (직접 대응)

1. 전사 일관성 — 에디터 구조 자체가 룰이므로 "규칙 + 센스" 조합이 사라진다.
2. 토큰 비용 — 변환 자체가 없어짐. Claude 호출은 "내용 초안 생성" 같은 선택적 보조로만.
3. 사소한 수정 — 브라우저에서 바로 고치고 저장. Claude Code 호출 불필요.

## 아직 열려 있는 것

- 블록 드래그 재정렬 UX 세부 — 잠정안은 노션식(블록 왼쪽 `⋮⋮` 핸들 + Cmd 클릭 멀티선택). Phase 3 구현 시 확정.
- 페이지 경계에 블록이 어색하게 걸리는 경우(박스 제목만 하단에 남고 내용은 다음 페이지, 그림과 캡션 분리, 리스트 첫 항목만 고립 등) 대응 필요성 — Phase 0 프로토타입으로 실사용 후 판단. 거슬리면 "이 블록 쪼개지 말기" 같은 블록 속성 추가 검토.
